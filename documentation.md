# Документация проекта «Система модерации объявлений Авито»

## Цель

Разработать веб-приложение для модерации объявлений на платформе Авито.

## Основной функционал

- Список объявлений с фильтрами, сортировками, пагинацией, множественным выбором и массовыми действиями.
- Карточка объявления с галереей, характеристиками, историей модерации и панелью действий (одобрить/отклонить/доработка).
- Статистика: метрики, графики, распределения, экспорт CSV/PDF, выбор периода.
- Горячие клавиши, тёмная тема, сохранение фильтров, симуляция новых объявлений.

## Структура

- `src/` — фронтенд (Vite + React + TS).
  - `pages/` — экраны (`ListPage`, `ItemPage`, `StatsPage`).
  - `shared/api/` — клиенты API (`ads`, `stats`, `moderator`) с `AbortSignal`.
  - `shared/utils/format.ts` — форматирование.
  - `shared/types.ts` — типы данных.
- `tech-int3-server/` — mock API сервер (Node/Express).
- `docker-compose.yml`, `Dockerfile` (клиент), `tech-int3-server/Dockerfile`, `nginx.conf` — окружение Docker.
- `.env` — `VITE_API_BASE` для фронта (локально).

## Фронтенд

- **TypeScript** - строгая типизация, меньше рантайм-ошибок по сравнению с JS, позволяет сделать продукт более управляемым, интересно было поработать с TS в данном проекте.
- **ESLint** - Предоставляет единый стиль и предотвращение ошибок, включая сервер.
- **Prettier** - автоформатирование кода для повышения читаемости и ускорения работы.
- **Vite** - быстрая разработка и сборка, оптимальная статика под nginx.
- **React Query** - предоставляет возможности и закрывает много задач: кэш, рефетч, управление состоянием запроса, отмена запросов через `AbortSignal` и т.д. В сравнении с обычным fetch или axios и UseState все пришлось бы писать вручную и тестировать, а в React Query уже все протестировано и собрано.
- **Tailwind CSS** - быстрая стилизация, краткость написания CSS стилей , простота применения, приятные встроенные элементы стилизации .

## Сервер и API

- `tech-int3-server/server.js` — Express, отдаёт mock-данные, поддерживает CORS (для фронта).
- Эндпоинты (схема в `tech-int3-server/schema.yaml`, README сервера):
  - `GET /api/v1/ads` — список объявлений (фильтры, пагинация).
  - `GET /api/v1/ads/:id` — детальная информация об объявлении.
  - `POST /api/v1/ads/:id/approve|reject|request-changes` — действия модерации.
  - `GET /stats/*` — метрики, графики, распределения, экспорт CSV/PDF, выбор периода.
  - `GET /moderators/me` — данные модератора.
- Вызовы на фронте всегда получают `AbortSignal` (React Query), чтобы отменять при навигации.
